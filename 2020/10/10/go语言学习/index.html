go中常用的命令

  1. go run -n 文件名.go

    打印编译过程所需运行的命令，但并不执行这些命令。

  2. go run -work 文件名.go

    显示编译时创建的临时工作目录的路径，并且不删除这个临时工作目录。

go编码规范

  1. 当命名 （常量、变量、类型、函数名、结构字段等等）以一个大写字母开头，使用这种形式的标识符的对象，就可以被外部包的代码所使用，这被称为导出。（类似于Java的public）
  2. 命名以小写字母开头，对包外不可见，但在整个包的内部是可见的并且可用的。（类似于Java的private）

defer

用于延迟函数的执行，如读写文件或者网络资源的时候需要关闭，可以先defer close()。

当外围函数的正常语句都执行完毕之后，只有其中所有的defer函数语句执行完毕后，外围函数才会真正的结束执行。

当外围函数return时，只有其中所有的defer函数执行完，外围函数才会真正返回。

当外围函数中的代码引发运行恐慌时，只有其中所有的延迟函数都执行完毕后，该运行时恐慌才会真正被拓展至调用函数。

如果存在多个defer语句，会按照栈的顺序执行，也就是逆序执行。
